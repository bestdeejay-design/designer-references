# Концепция «Справочника дизайнера»

> Дополнение к `README.md`. Здесь — не «как запустить», а **зачем** этот
> проект существует, из чего состоит и в чём его настоящий смысл.

---

## 1. Цели

Проект родился как вынос куска из портфолио Ксении (`ksu/`): там
уже был собран пласт референсов, и стало ясно, что он живёт своей
жизнью, отдельно от «витрины портфолио».

**Главные цели:**

- **Снять творческий ступор.** Когда голова пуста, нужен не совет,
  а *вход*. Открыл — и сразу видишь десятки направлений и живых
  примеров. Это кран, а не лекция.
- **Быть компасом, а не свалкой.** Каждая карточка ведёт не «в никуда»,
  а на конкретный источник (Behance / Dribbble / Figma / Pinterest) или
  на сервис мудбордов (Are.na / Muzli / Designspiration). Справочник
  отвечает на вопрос «куда идти за вдохновением», а не «что посмотреть
  прямо тут».
- **Дать быстрый шеринг заказчику.** Вместо ссылок по кусочкам в
  переписке — одна подборка по категориям, одна ссылка.
- **Сохранить накопленный вкус.** Референсы из `ksu` — это не просто
  закладки, а *отпечаток глаза* дизайнера. Вынос их в отдельный проект
  превращает личный опыт в переиспользуемый инструмент.

## 2. Структура

```
references/
├── index.html        точка входа: собирает страницу, задаёт lang, токены темы
├── references.css    стили блока References (.references, .ref-card*, .ref-mini*)
├── references.js     данные + функции рендера (buildReferences / buildResourceSection)
├── README.md         техническая документация (запуск, структура, план)
└── docs/
    └── CONCEPT.md    эта концепция (цели / структура / возможности / смысл)
```

**Модель данных (`references.js`):**

| Сущность | Назначение |
|---|---|
| `refsI18n` | переводы заголовков/подзаголовков, ключи `refs.*` (en + ru) |
| `refsData` | основные направления: `{ cat, desc, behance, dribbble, figma, pinterest, color }` для en и ru |
| `specialResources` | спецресурсы (It's Nice That, Awwwards, Typewolf, …) |
| `boardResources` | сервисы мудбордов (Are.na, Muzli, Designspiration) |

**Принцип сборки.** `index.html` — «оживляющая» обвязка. Он:
1. подключает `references.css` + `references.js`;
2. определяет глобальную `lang` (её читают функции рендера);
3. задаёт CSS-токены темы локально (в оригинале они шли из
   `ksu/css/tokens.css`, здесь продублированы, чтобы проект был
   самодостаточным);
4. рисует три сетки: `#references-grid`, `#special-grid`, `#boards-grid`.

**Два переключателя в шапке:** язык **RU / EN** и тема **тёмная / светлая**
(через атрибут `data-theme` на `<html>`).

## 3. Возможности

**Уже работает:**
- билингва RU/EN с мгновенным пересбором всех сеток;
- тёмная и светлая темы;
- три типа блоков: направления (18 категорий), спецресурсы (5),
  сервисы мудбордов (3);
- прямые ссылки на Behance / Dribbble / Figma / Pinterest по каждой
  категории + отдельные карточки ресурсов/сервисов.

**В планах (см. README, раздел «Что дальше»):**
1. **Категории** — навигация по направлениям из `refsData`
   (брендинг, упаковка, плакаты, иллюстрация, 3D, …).
2. **Быстрый шеринг заказчику** — выбрать несколько карточек по
   разным проектам и сгенерировать одну ссылку-подборку.
3. **Фильтр по проектам** — показывать релевантные референсы
   под конкретную задачу заказчика.
4. **Дизайн-система** — переиспользовать токены из `ksu/css/tokens.css`,
   чтобы справочник был в одном стиле с портфолио.

**Среда исполнения.** Проект статический, сборка не нужна. Проверка
рендера выполняется в изолированном контейнере (OrbStack / Docker +
Playwright), без зависимости от локального браузера.

## 4. Сокровенный смысл

На поверхности — подборка ссылок для дизайнера. Но если копнуть,
проект про другое.

**Это акт кураторства, а не агрегации.** Бесконечная лента убивает
вкус объёмом. Здесь — конечный, *отобранный* набор. Граница —
осознанный выбор: «из этого я советую смотреть вот это». Справочник
делает вкус видимым и передаваемым.

**Это перевод с языка «внутри» на язык «заказчику».** Референсы
выросли из портфолио `ksu` — из накопленного внутреннего чутья.
Но адресованы они и самому дизайнеру (когда нет идей), и клиенту
(когда нужно согласовать направление). Это мост между интуицией
автора и потребностью другого человека — без потери лица.

**Это компас, а не образец.** Каждая карточка говорит «иди туда»,
а не «скопируй это». В эпоху генеративных однообразных мудбордов
человеческая, предвзятая, *именная* карта источников — тихий протест
против усреднения. Она предполагает оригинальность, а не подражание.

**Это внешняя память о том, что кажется красивым.** Справочник —
своего рода дворец памяти вкуса: когда забываешь, «чего ты вообще
хотел», он напоминает. Не фактами, а направлениями.

В итоге «Справочник дизайнера» — это не инструмент подбора картинок.
Это **привычка смотреть осмысленно** и **способность делиться
своим видением**, не разменивая его на бесконечный скролл.
