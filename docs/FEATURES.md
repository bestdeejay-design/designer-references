# Фичи, практики, дифференциация и аудитория

> Документ-стратегия проекта «Справочник дизайнера».
> Дополняет `docs/CONCEPT.md` (цели/структура/смысл).
> Здесь — конкретнее: что умеем сейчас, что можно добавить,
> какие практики лежат в основе, чем выделяемся и кому нужно.

Проект — **чисто визуальный справочник + путеводитель «куда идти за
референсами»**. Вырезан из портфолио `ksu/` (референсы уже были
собраны там как структурированный гид), положен в отдельный проект.
Статический сайт, сборка не нужна, публикуется на GitHub Pages.

---

## 1. Что уже умеет (MVP — есть сейчас)

- **18 направлений дизайна** с прямыми ссылками на 4 площадки
  (Behance / Dribbble / Figma / Pinterest) по каждому направлению.
- **Две темы** (тёмная / светлая) и **два языка** (RU / EN) — переключение
  мгновенное, без перезагрузки.
- **Живой поиск по всем секциям**: направления + спецресурсы + сервисы
  мудбордов. Описания расширены, чтобы поиск находил «брендбук»,
  «винил», «мудборд», «UI/UX» и т.д.
- **Секция спецресурсов** (It's Nice That, Design Week, Awwwards,
  SiteInspire, Typewolf) и **сервисов мудбордов** (Are.na, Muzli,
  Designspiration).
- **Нулевая сборка**: статический HTML/CSS/JS, открывается двойным
  кликом или любым локальным сервером.
- **Публичный URL** на GitHub Pages — ссылку можно дать клиенту.

## 2. Что можно сделать (дорожная карта / бэклог)

Сгруппировано по смыслу. ✅ — есть; ⬜ — в планах.

### А. Навигация и структура
- ✅ Живой текстовый поиск по всем секциям.
- ✅ **Кнопки-категории** (Брендинг / Упаковка / Графика / Реклама /
   Цифра / Арт + «Все») — быстрый переход к группе направлений.
- ⬜ **Теги / древо направлений** (например, «упаковка алкоголя»,
  «детский бренд», «премиальный веб») — фильтр тоньше, чем по тексту.
- ✅ **Избранное у дизайнера** — сохранять частые направления/ссылки
   локально (localStorage), чтобы не искать заново. Выбор персистится
   между сессиями; `?sel=` (чужая подборка) имеет приоритет при открытии.

### Б. Шеринг с клиентом (ключевая фича проекта)
- ✅ **Подборка-ссылка**: выбрать несколько карточек по разным проектам
   → сгенерировать одну ссылку (`?sel=...`). Клиент открывает — видит
   только отобранное.
- ✅ **Режим презентации**: подборка на весь экран, без лишнего UI
   (← → листать, Esc выйти; заливка карточки из её токена).
- ⬜ **Комментарии к карточкам** внутри подборки (что именно нравится).
- ✅ **Экспорт подборки** в сообщение (markdown-список ссылок)
   одной кнопкой («Список» в панели подборки).

### В. Умный поиск и рекомендации
- ⬜ **Визуальный поиск** (по близости картинок, а не по тексту) —
  через внешние API Pinterest/Behance или локальный эмбеддинг.
- ⬜ **«Похожее»**: по выбранной карточке — родственные направления.
- ⬜ **Фильтр по платформе / по цвету / «только свежие»**.

### Г. Дизайн-система и полировка
- ✅ **Переиспользовать токены** из `ksu/css/tokens.css` — палитра,
   интервалы и бренд-акцент (`#FF2D55`) выровнены под портфолио,
   токены остаются локальными в `index.html` (проект самодостаточен).
- ✅ **Адаптив под мобильные**: сетка 2→1 колонка, контейнеры
   на `max-width`+центр, чипы переносятся, нет горизонтального
   переполнения (проверено на 375 / 768 / 1440).
- ✅ **Доступность**: видимый фокус `:focus-visible` акцентом
   на всех интерактивных элементах (поиск, кнопки, чипы, ссылки).
- ⬜ **PWA / оффлайн** — кэш ссылок, чтобы открывать без сети.

### Д. Расширение контента
- ⬜ **Хранить данные в отдельном файле** (не в JS-массиве внутри
  страницы) — проще добавлять направления и ревью через PR.
- ⬜ **Импорт референсов** из Pinterest-доски или Are.na-канала одной
  кнопкой.
- ⬜ **Ещё языки** через уже заложенный механизм i18n.

## 3. Практики (как сейчас собирают и шарят референсы)

Реальные привычки, на которые опирается проект:

- Дизайнеры держат «банки вдохновения» в **Pinterest / Are.na / Muzli /
  закладках браузера**. Это личные кладовки, не приспособленные для
  показа клиенту.
- В переписке с заказчиком ссылки обычно кидают **кусками** —
  скриншоты в Telegram, по 1–3 ссылке. Теряется контекст, трудно
  сравнивать, легко уйти в спор «я не понял, чего ты хочешь».
- Студии делают **moodboard** в Figma / Notion / Canva — но это
  внутреннее, клиенту неудобно листать, а поддерживать — долго.
- Главная проблема: **нет единого «путеводителя куда идти»**, особенно
  для не-дизайнеров. Клиент не знает, что вообще бывает в упаковке,
  брендинге или вебе, поэтому не может сформулировать вкус.
- Наша практика (из `ksu/`): референсы уже были собраны в портфолио как
  **структурированный гид по направлениям**. Мы вынесли это в отдельный
  инструмент — и получили не «закладки», а именно навигатор.

## 4. Чем выделяемся (дифференциация)

- **Не «ещё один менеджер закладок» и не «ещё одна доска Pinterest».**
  Это **путеводитель по направлениям** с готовой структурой
  «куда идти за референсом» — по 4 площадкам сразу.
- **Клиенто-ориентированность.** Заточен под то, чтобы дизайнер за
  минуту собрал клиенту подборку и скинул **одну ссылку** вместо
  простыни ссылок в чате. (Это и есть «сокровенный смысл» проекта —
  см. `docs/CONCEPT.md`.)
- **Минимум текста, максимум превью и ссылок.** Чисто визуально —
  как и должен выглядеть справочник дизайнера.
- **Нулевая сборка + GitHub Pages.** Открывается где угодно, бесплатно,
  без бэкенда и регистрации.
- **Двуязычность RU/EN из коробки** — редкость для таких гидов.
- **Честная структура из реального портфолио** (`ksu/`), а не
  сгенерированный список «топ-100 сайтов».
- Потенциально — **брендированный гид студии**: клиент видит не «чужой
  Pinterest», а фирменный справочник конкретного дизайнера.

## 5. Кому интересен и почему будут пользоваться

### Персоны

1. **Фриланс-дизайнеры и малые студии**
   - *Зачем*: не собирать референсы с нуля под каждый бриф и быстро
     согласовывать вкус с клиентом.
   - *Почему пользуются*: экономит время на онбординг заказчика и снимает
     «я не знаю, чего хочу».

2. **Клиенты-недизайнеры** (предприниматели, маркетологи, основатели
   стартапов)
   - *Зачем*: понять, как выглядит «упаковка / брендинг / сайт» и выбрать
     направление без дизайн-образования.
   - *Почему пользуются*: снижает порог входа, даёт общий словарь с
     исполнителем.

3. **Студенты и начинающие дизайнеры**
   - *Зачем*: как учебный гид «куда смотреть за вдохновением».
   - *Почему пользуются*: структура из реального портфолио = проверенные
     источники, а не шум ленты.

4. **Дизайн-студии (внутреннее использование)**
   - *Зачем*: библиотека направлений для брифинга команды и клиентов.
   - *Почему пользуются*: единый источник правды, можно брендировать под
     студию.

5. **Преподаватели / курсы по дизайну**
   - *Зачем*: как раздатку «обзор направлений».
   - *Почему пользуются*: компактно и наглядно, сразу с рабочими ссылками.

### Почему возвращаются
- **Быстрый доступ**: «нет идей → открыл → увидел десятки направлений».
- **Единая ссылка** вместо простыни ссылок в чате.
- **Бесплатно, без регистрации, работает везде**.
- **Структура растёт вместе с практикой** дизайнера (добавляем
  направления под новые задачи).
