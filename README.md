# Справочник дизайнера

> Отдельный проект, выделенный из портфолио Ксении (`ksu/`).
> Основа — референсы, которые уже были собраны в портфолио
> (см. `references.js` и `references.css` в этой папке).

Чисто визуальный справочник + путеводитель «куда идти за референсами».
Никакой лишней теории — только картинки, ссылки и быстрый доступ.

---

## Когда нужен

- **Нет идей.** Застой, выгорел, пустая голова — открыл и сразу
  увидел десятки направлений и живых примеров.
- **Надо быстро скинуть заказчику референсы по нескольким проектам
  сразу.** Вместо того чтобы собирать ссылки по кусочкам в переписке,
  формируешь подборку по категориям и отправляешь одной ссылкой.

## Принципы

- Проект **чисто визуал**: минимум текста, максимум превью и ссылок.
- Это **справочник куда идти**: каждая карточка ведёт на реальный
  источник (Behance / Dribbble / Figma / Pinterest) или на сервис
  для мудбордов (Are.na / Muzli / Designspiration).
- База — референсы из портфолио `ksu`. Их можно дополнять и
  расширять под новые задачи.

---

## Структура проекта

```
references/
├── index.html        ← точка входа: собирает страницу, подключает css+js
├── references.css    ← стили блока References (.references, .ref-card*, .ref-mini*)
├── references.js     ← данные (refsData, specialResources, boardResources) + рендер
├── README.md         ← техническая документация (запуск, структура, план)
└── docs/
    └── CONCEPT.md    ← концепция: цели, структура, возможности, смысл
```

`index.html` — это «оживляющая» обвязка, которой не хватало исходному
вырезанному коду. Он:

1. подключает `references.css` и `references.js`;
2. задаёт глобальную переменную `lang` (её читают функции рендера);
3. определяет CSS-токены темы (`--bg`, `--text`, `--border`, …) —
   они приходят из `ksu/css/tokens.css` в оригинале, здесь заданы
   прямо в `<style>` страницы, чтобы проект был самодостаточным;
4. рисует шапку и три сетки: `#references-grid`, `#special-grid`,
   `#boards-grid`.

## Как запустить

Проект статический, сборка не нужна.

- **Просто открой** `index.html` в браузере (двойной клик / `file://`).
- **Или подними локальный сервер** (рекомендуется, чтобы не блокировались
  некоторые браузерные политики):

  ```bash
  # из папки references/
  python3 -m http.server 8000
  # затем открой http://localhost:8000
  ```

> Шрифт `Unbounded` подгружается с Google Fonts; при оффлайне
> страница корректно отрисуется системным шрифтом.

## Как это работает

- **Язык (RU / EN).** Кнопки в шапке переключают глобальную переменную
  `lang` и пересобирают все сетки. Тексты берутся из `refsI18n`.
- **Тема (тёмная / светлая).** Кнопка ☾/☀ переключает атрибут
  `data-theme` на `<html>`; стили для светлой темы заданы через
  `[data-theme="light"]` в `references.css`.
- **Сетки.** `buildReferences()` заполняет `#references-grid` карточками
  направлений; `buildResourceSection(id, items)` — `#special-grid` и
  `#boards-grid` списками ресурсов.

## Модель данных (`references.js`)

| Переменная | Назначение |
|---|---|
| `refsI18n` | Переводы заголовков/подзаголовков (en + ru), ключи `refs.*` |
| `refsData` | Основные референсы по направлениям: `{ cat, desc, behance, dribbble, figma, pinterest, color }` для en и ru |
| `specialResources` | Спецресурсы (It's Nice That, Awwwards, Typewolf, …) |
| `boardResources` | Сервисы мудбордов (Are.na, Muzli, Designspiration) |

Чтобы добавить направление — допиши объект в `refsData.en` / `refsData.ru`.
Чтобы добавить ресурс — в `specialResources` / `boardResources`.
